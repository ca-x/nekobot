## Design Context

### Users
Operators managing AI runtimes, sessions, tools, providers, prompts, and workspace state from a web dashboard. Primary usage context is desktop operations, but key flows must remain usable on mobile/tablet for quick inspection, approval, and lightweight control tasks.

### Brand Personality
Operational, trustworthy, quietly polished.

### Aesthetic Direction
A glassy, modern control-surface UI with warm/premium accents and strong readability. Keep the dashboard feeling technical and composed rather than playful. Existing auth pages use stronger branding; main workspace should stay information-dense and calm.

### Design Principles
1. Preserve operator clarity over visual flair.
2. Make narrow screens usable without hiding core actions.
3. Prefer stacked, progressive layouts on mobile over cramped multi-column compression.
4. Keep touch targets and controls comfortably tappable.
5. Maintain consistent shell/navigation behavior across screen sizes.
